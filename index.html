ssh服务

SSH服务端主要提供两个服务的功能：

  • 类似telnet的远程连接服务
  • 类似FTP服务的sftp-server，借助SSH协议来传输数据，提供更加安全的SFTP服务。

SSH客户端有一个重要的命令scp

在Linux上运行着ssh的服务端

公钥和私钥的概念

公钥

公钥（public key）：提供给远程主机进行数据加密，任何人都能获得的公钥来加密数据

私钥

私钥（private key）：远程主机通过本机的公钥进行的加密的数据，在本机就需要私钥进行解密。因此私钥很重要

SSH服务连接过程

  • 服务端建立公钥文件，当SSH服务启动时，会产生一个786-bit的临时公钥，存放在/etc/ssh/ssh_host*文件中
  • 客户端发起连接请求，服务端会将生成好的公钥文件发送到客户端，客户端如果不是第一次连接，则会比对此公钥和原先存储的公钥是否相同，如果是第一次连接，则会将公钥存储在~/.ssh/known_hosts文件中

Linux命令-sed的艺术（一）

发表于 2017-08-10

sed（stream editor for filtering and transforming text）是一种编辑器，全名是流式编辑器（Stream Editor），可以基于命令来操作数据流中的数据，这些命令要么从命令行中输入，要么保存在文本中。sed每次从输入中读取一行进行处理，并用提供的编辑器命令匹配该数据，按照命令中指定的操作修改数据，将生成的结果输出（stdout）。sed是对文件的行处理，如果没有file参数，sed会对标准输入作为处理。

sed的scripts是对匹配上的 行做的处理。可以把sed看作一个编辑器，是一种更高级的编辑器。只不过sed是面向数据流编辑的，看不见文本的内容。sed的工作原理：sed每次一次处理文件的一行，处理时会将内容存放在临时缓冲区（学名：模式空间），处理完成后会将数据的内容送完屏幕显示出来。接着处理下一行。但是文件的内容只是看起来被修改了，实际文件的内容并没有被修改，如果想要模式空间的内容修改文件，需要加上修改参数，来修改文件的实际内容。

a
b
c
c    #c打印了两次 p代表打印匹配上的行    
[root@1 sedDemo]# sed -n '/c/p' a.txt   #加上了-n参    数，代表只想打印匹配的行
c

sed最重要的就是sed-scripts，sed默认对文件内容的处理方式就是，将模式空间的内容输出到屏幕上。除了使用d标志来删除一行，否则文件内容都会在屏幕上显示。
