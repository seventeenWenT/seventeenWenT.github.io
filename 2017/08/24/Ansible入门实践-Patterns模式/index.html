Ansible的入门实践

Ansible的Patterns是用来寻找主机。类似找谁，干什么的格式
命令的格式：

ansible <pattern_goes_here> -m <module_name>s -a <arguments>

通过pattern来寻找要执行命令的主机
类似：

[root@mananger /]# grep "model02" /etc/ansible/hosts  -C4
[webserver]
10.0.0.17[2:3]
[model02]
10.0.0.172

执行命令：

[root@mananger /]# ansible model02 -m ping
10.0.0.172 | SUCCESS => {
"changed": false,
"ping": "pong"
}

无论如何都要告诉ansible要指定给那台主机执行命令

指定*或者all代表所有机器

通过指定IP地址来来进行管理

[root@mananger /]# ansible 10.0.0.172 -m ping
10.0.0.172 | SUCCESS => {
"changed": false,
"ping": "pong"

}

通过！来排除某些组

通过：来分隔多个组

[root@mananger /]# ansible model01:model02 -m ping
10.0.0.172 | SUCCESS => {
"changed": false,
"ping": "pong"
}
10.0.0.173 | SUCCESS => {
"changed": false,
"ping": "pong"
}

可以通过正则表达式来匹配，只不过正则表达式要用~开头

[root@mananger /]# ansible ~model[0-9]{2} -m ping
10.0.0.172 | SUCCESS => {
"changed": false,
"ping": "pong"
}
10.0.0.173 | SUCCESS => {
"changed": false,
"ping": "pong"
}
